## 6\. 页面 UI 详细设计

### 6\.1 团队端：看板页面

完全复用参考 UI 的布局与交互：

1. 看板列支持自定义增删改，适配不同项目的流程

2. 任务卡片支持拖拽跨列流转，拖拽时有实时的位置反馈

3. 点击任务卡片或者新建按钮，弹出右侧的表单抽屉，和参考图的交互完全一致

4. 任务卡片展示标题、负责人头像、优先级标签，hover 时展示更多操作按钮

### 6\.2 客户端：进度门户页面

针对客户的只读场景做简化：

1. 去掉左侧导航与复杂操作栏，仅保留项目名称与进度概览

2. 看板列和团队端保持一致，但是去掉所有编辑、拖拽的交互，完全只读

3. 顶部突出展示项目里程碑，让客户快速了解整体交付阶段

4. 自动适配移动端，客户在手机上也能流畅查看

### 6\.3 数据统计页面

1. 顶部核心指标卡片：展示任务完成率、平均周期时间、吞吐量、在制品数量四个核心指标，用大字体突出展示

2. 图表区域：

    - 累积流图：用 Recharts 实现，展示各状态列的任务数量变化

    - 燃尽图：展示迭代的工作量变化

    - 成员产能卡片：展示团队成员的任务完成情况

3. 所有图表支持时间范围筛选，支持按周 / 月查看数据

## 7\. 交互设计

1. **拖拽交互**：拖拽任务卡片时，卡片有轻微的缩放与阴影效果，目标列有高亮提示，释放后有平滑的位置过渡动画

2. **抽屉动画**：新建 / 编辑任务的抽屉，从右侧平滑滑入，关闭时平滑滑出，过渡时间 200ms

3. **状态变更反馈**：任务状态修改后，有轻微的成功提示，无需页面刷新

4. **空状态处理**：当看板没有任务时，展示友好的空状态提示，引导用户新建任务

## 8\. 响应式适配

1. **桌面端（\&gt;1024px）**：完整的四栏布局，和参考图完全一致

2. **平板端（768px\-1024px）**：左侧导航自动折叠为可展开的菜单，看板列支持横向滚动
